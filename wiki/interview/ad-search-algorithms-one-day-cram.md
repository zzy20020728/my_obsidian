# 广告搜索算法 1 天速背版

> 用法：只背这份也能撑起几轮面试。目标不是精通，而是先建立稳定答题框架。

## 第一部分：先背全局框架

广告搜索算法主链路：

Query Understanding
-> Recall
-> Pre-rank
-> Rank
-> Auction / Re-ranking
-> 展示
-> 点击/转化反馈

一句话解释：

> 本质上是在强时延约束下，从海量候选里找出既相关、又有收益、又符合业务规则的结果。

## 第二部分：必须会说的 8 个概念

### 1. Recall / Pre-rank / Rank

- Recall：保上限，别漏掉候选
- Pre-rank：轻量筛选，控成本
- Rank：重模型决定最终质量

### 2. CTR / CVR / eCPM

- CTR：点击率
- CVR：转化率
- eCPM：千次展示收益

### 3. GSP

广义二价拍卖，排序看综合价值，扣费参考下一名临界价格。

### 4. OCPX

从点击优化扩展到转化优化，让系统自动围绕转化目标调出价。

### 5. AUC / LogLoss / GAUC / NDCG

- AUC：排序区分能力
- LogLoss：概率是否准
- GAUC：分组场景下更合理
- NDCG：位置质量

### 6. Position Bias

前排广告天然更容易被点，点击不完全等于真实相关性。

### 7. Calibration

广告里预测概率会直接参与 auction 和收益计算，所以概率必须准。

### 8. 多目标优化

广告排序不是只看一个指标，而是用户体验、平台收益、广告主 ROI 的平衡。

## 第三部分：最常见的 10 个标准答法

### Q1：召回、粗排、精排分别做什么？

答：召回负责高覆盖，粗排负责低成本筛选，精排负责最终质量。工业系统一定是分阶段漏斗，不可能全量精排。

### Q2：为什么 Reranker 不能解决所有问题？

答：因为它只能排已有候选，不能补召回。系统上限首先由 query understanding 和 recall 决定。

### Q3：为什么广告排序是多目标优化？

答：因为要同时兼顾相关性、收益、广告主 ROI 和生态约束，不能只追一个指标。

### Q4：为什么不能只按 bid 排？

答：因为会损伤用户体验，高出价但低相关的广告也会占位。

### Q5：为什么 calibration 很重要？

答：因为预测概率会直接用于出价、收益和预算控制，所以不仅要排得对，还要估得准。

### Q6：GSP 和 OCPX 的区别？

答：GSP 更经典，偏点击广告和二价拍卖逻辑；OCPX 更偏转化优化，目标更深。

### Q7：AUC 和 LogLoss 区别？

答：AUC 看排序，LogLoss 看概率。广告里两者都重要。

### Q8：为什么搜索广告一定要低时延？

答：因为它是高 QPS 在线系统，晚几十毫秒都可能影响体验和收益，所以架构上一定是分层漏斗和缓存优先。

### Q9：如果没做过广告，你怎么迁移？

答：我做过 query understanding、召回、融合、重排和服务化，这些是广告搜索算法的主链路。我要重点补的是 auction、预算、频控和 ROI 机制层。

### Q10：你现有项目和广告搜索的共性是什么？

答：共性在于都是 query 到 recall 到 rank 的链路。我的项目虽然是学术检索和 RAG 场景，但我实际做过 query planning、多源召回、RRF 融合和 rerank，这些方法论是可以迁移的。

## 第五部分：最容易被追问的点

### 1. 你没做 auction，怎么办？

答：承认。不要装懂。说自己主干能力在搜索和排序，auction 是正在重点补的机制层。

### 2. 你没做线上大规模系统，怎么办？

答：说自己做过服务化、检索链路和多阶段排序思路，对高 QPS 系统的核心约束有理解，但具体广告大规模业务细节还在补。

### 3. 你不会广告业务黑话怎么办？

答：先把这几个词背熟：CTR、CVR、eCPM、GSP、OCPX、ROI、预算、频控、探索、校准、多目标优化。
