# 广告搜索算法高频 Q&A

## 1. 广告搜索算法的整体链路是什么？

答：通常是 Query Understanding -> Recall -> Pre-rank -> Rank -> Auction / Re-ranking -> 展示 -> 点击/转化反馈。前半段解决“找什么、排什么”，后半段解决“怎么兼顾收益和体验”。

## 2. 召回、粗排、精排分别做什么？

答：召回负责从海量候选里尽量别漏掉；粗排负责用轻量模型快速缩小候选集；精排负责用更重模型做最终排序。工业系统不可能全量精排，所以必须分阶段漏斗。

## 3. 为什么不能直接对全量广告做精排？

答：因为计算成本和时延都不允许。广告/搜索系统通常是高 QPS、低时延场景，必须先便宜筛选，再昂贵精排。

## 4. CTR、CVR、CTCVR 分别是什么？

答：CTR 是点击率，CVR 是点击后的转化率，CTCVR 是展示到转化的整体概率，常可近似理解为 CTR 和 CVR 的联动目标。

## 5. eCPM 是什么？

答：eCPM 是每千次展示的预估收益。在最经典的点击广告排序里，可以近似理解为 `bid * pCTR * 1000`。如果优化更深层目标，还会加入 pCVR 或转化价值。

## 6. 广告排序为什么是多目标优化？

答：因为广告排序同时服务三方目标：用户体验、平台收益、广告主 ROI。不能只看相关性，也不能只看收益。

## 7. 搜索和广告排序的主要区别是什么？

答：搜索更强调相关性和 query 意图；广告排序在相关性基础上还要考虑 bid、CTR、CVR、预算、频控、收益和生态约束。

## 8. GSP 是什么？

答：GSP 是 Generalized Second Price，广义二价拍卖。思想是排序按综合价值决定，但扣费参考下一名的临界价格，而不是自己申报的最高价。

## 9. OCPX / OCPA 是什么？

答：它们是从点击目标扩展到转化目标的自动出价机制。系统不只优化点击，还会根据预估转化概率帮广告主调整出价，更偏结果导向。

## 10. 广告排序分数一般怎么设计？

答：一个经典形式是 `score = bid * pCTR`。如果更关注转化，可以扩展成 `bid * pCTR * pCVR * value`。真实系统里还会加相关性约束、预算和频控等规则。

## 11. 为什么不能只按 bid 排？

答：如果只按 bid 排，会损害用户体验，因为高出价但低相关的广告也会排到前面。广告系统必须在相关性和收益之间平衡。

## 12. 为什么不能只按 CTR 排？

答：因为平台还要考虑收益和广告主商业目标。单看 CTR 可能点击高但收益低，也可能忽略高价值转化广告。

## 13. 为什么 calibration 很重要？

答：因为 CTR/CVR 预测结果不只是排序分数，它们还会被用于出价、收益估计和预算控制。如果概率不准，会直接影响 auction 和 ROI。

## 14. AUC 和 LogLoss 分别看什么？

答：AUC 更看模型的排序区分能力；LogLoss 更看概率预测是否准确。广告里两者都重要，因为既要排得对，也要估得准。

## 15. GAUC 为什么常用于广告排序？

答：因为广告/搜索数据天然是按用户、query 或 session 分组的。GAUC 在组内衡量排序能力，通常比全局 AUC 更接近真实线上场景。

## 16. NDCG 什么时候常见？

答：搜索场景更常见，因为它显式考虑了位置权重，越靠前位置越重要。广告搜索排序里也会用它来评估位次质量。

## 17. 广告排序里为什么会有 position bias？

答：因为排在前面的广告天然更容易被看到和点击，所以点击数据本身就受位置影响，不是纯粹由广告质量决定的。

## 18. 什么是 selection bias？

答：只有被系统展示出来的广告才有点击和转化数据，所以训练样本是经过旧策略筛选过的，不是全空间的随机样本。

## 19. 这些偏差怎么处理？

答：常见方法包括 IPS/DR、debias、校准、位置特征建模、随机探索、在线实验和样本重加权。

## 20. 冷启动怎么做？

答：新广告可以依赖内容特征、类目特征、相似广告迁移；新用户可以依赖上下文、地域、设备和 query；新 query 可以依赖同义词扩展、向量召回和规则 fallback。

## 21. 搜索里的 query understanding 包括哪些内容？

答：分词、纠错、同义词扩展、query rewrite、意图识别、类目理解、实体识别等。它直接影响召回上限。

## 22. 为什么说召回决定上限？

答：因为排序只能在已有候选里选最优。如果相关广告根本没被召回进来，再强的 ranker 也没法把它排上去。

## 23. Dense recall 和 sparse recall 各自优缺点是什么？

答：sparse 更擅长关键词精确匹配、可解释性强；dense 更擅长语义泛化。真实系统里常把两者结合做 hybrid recall。

## 24. RRF 为什么常用于融合？

答：因为不同召回路的分数分布不稳定、不可比，RRF 只依赖 rank，不强依赖原始分值，所以融合更稳。

## 25. Reranker 的价值是什么？

答：它在较小候选集上做更细粒度的相关性判断，提升最终排序质量。

## 26. Reranker 的边界是什么？

答：它不能补召回。这个是面试里非常值得强调的一点。

## 27. LR 为什么在广告里经典？

答：因为简单、稳定、可解释，对高维稀疏离散特征友好，训练和上线成本低，是工业界非常经典的 baseline。

## 28. FM / DeepFM 为什么常见？

答：因为广告场景里高维 sparse 特征很多，特征交叉非常关键。FM 适合显式二阶交互，DeepFM 同时建模低阶和高阶交互。

## 29. DIN 为什么常被问？

答：因为用户兴趣是动态的，DIN 通过 attention 从历史行为中选择和当前候选最相关的兴趣片段，非常适合 CTR 预估。

## 30. 多任务学习为什么常用于广告？

答：因为 CTR、CVR、停留时长、跳失率、深层转化等目标相关但不完全相同。共享底座 + 多任务头可以提升样本利用率，但要处理任务冲突。

## 31. MMOE / PLE 为什么会被问？

答：它们是多任务学习里的经典结构，用来缓解共享表示不足和任务冲突问题。面试官常想看你是否理解 shared-bottom 到 expert-routing 的演进逻辑。

## 32. 特征工程在广告里有哪些核心类别？

答：用户特征、广告特征、query 特征、上下文特征、交叉特征、统计特征、时序行为特征。

## 33. 广告里为什么常做交叉特征？

答：因为点击与否通常不是单边属性决定的，而是 user-ad、query-ad、user-query-ad 的交互结果。

## 34. 为什么离线指标高，线上不一定好？

答：因为线上还有 auction、预算、频控、探索、位置偏差、数据漂移和系统约束，离线只反映模型的一部分能力。

## 35. 线上实验最重要看什么？

答：通常同时看 CTR、CVR、GMV / Revenue、用户体验指标、广告主 ROI、时延和系统稳定性，而不是只看单一指标。

## 36. 为什么时延在搜索广告里很重要？

答：因为这类系统请求量大、用户敏感度高，晚几十毫秒都可能影响体验和收益。所以工程上一定是分层漏斗、缓存和轻重模型配合。

## 37. 常见缓存有哪些？

答：倒排/ANN 索引缓存、特征缓存、embedding 缓存、结果缓存、模型输出缓存等。

## 38. 为什么 feature consistency 很重要？

答：如果离线训练和线上推理的特征不一致，模型效果会严重打折。真实工程里要强调离在线同构和特征平台一致性。

## 39. 如果面试官问“你没做过广告，为什么能做广告搜索算法？”

答：我的项目场景虽然不是广告，但底层方法论是相通的。我做过 query understanding、召回、融合、重排和服务化，这些正是搜索广告算法的主干。广告方向我需要补的是 auction、预算、频控、ROI、多目标优化这些机制层，但算法主链路我已经有对应实践基础。

## 40. ScholarLens v2 怎么迁移到广告搜索表达？

答：它本质上是 query -> 多源召回 -> 融合 -> rerank -> 迭代优化 的搜索链路。我做过 QueryAgent、RRF、Cross-Encoder、服务化 API，这些都可以迁移到广告搜索里的 query understanding、recall 和 ranking。

## 41. Agentic RAG 项目怎么迁移到广告搜索表达？

答：我重构过 chunk-based ingestion、dense+sparse hybrid retrieval、RRF 融合、reranker 和结构化 workflow。这些和广告搜索系统里的索引粒度设计、混合召回、排序链路和工程稳定性是同构的。

## 42. 如果问你最熟悉哪一块，你怎么选？

答：优先说搜索主链路，也就是 query understanding、recall、fusion、rerank 和服务化。因为这部分你有真实项目经验，比硬讲 auction 更稳。

## 43. 如果问“广告里最难的是什么？”

答：我会说最难的是多目标优化和平衡。因为广告系统既要保证相关性和体验，又要考虑收益、广告主 ROI、预算和生态健康，不是单一模型问题。

## 44. 如果问“你最想补的广告方向能力是什么？”

答：我会说机制层，包括拍卖、预算控制、频控、探索利用和平滑放量，因为这是我目前业务经验里相对薄弱但最有价值的一层。

## 45. 如果问“你做的项目和广告最大的差异是什么？”

答：最大的差异不是检索和排序主链路，而是机制层。我的项目更多是搜索/检索系统，广告还要在此基础上叠加 bid、ROI、预算和 auction。

## 46. 如果问“你觉得广告和推荐的共性是什么？”

答：都要做候选召回、排序、多目标优化和反馈闭环。差异在于广告更强商业约束，搜索更强 query 驱动。

## 47. 如果问“你觉得广告搜索为什么比普通搜索更难？”

答：因为普通搜索更多是相关性问题，而广告搜索要在相关性、收益、广告主目标和平台规则之间做平衡，机制层复杂得多。

## 48. 如果问“你会怎么快速上手广告搜索业务？”

答：我会先把系统链路拆成 query understanding、recall、ranking、auction 和 online experiment 五层。前两三层我有可迁移经验，后两层重点补业务机制和线上策略。

## 49. 如果问“广告算法岗最重要的能力是什么？”

答：我觉得是两种能力结合：一是检索/排序/预估模型能力，二是对业务机制和线上系统约束的理解。只会建模但不理解 auction 和策略，很难落到真实广告系统里。

## 50. 最后怎么总结自己和这个方向的匹配度？

答：我的强项在搜索和排序主链路，包括 query understanding、召回、融合、重排和系统工程化；广告方向我已经在补 auction、多目标优化和业务指标体系。我不是从零开始转方向，而是在已有检索排序基础上往广告机制层补齐。
