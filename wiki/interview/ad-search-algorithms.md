# 广告搜索算法面试速成手册

> 目标：快速补齐广告/搜索算法岗最常问的知识框架，并把你现有项目映射到这个方向上。

## 一句话理解这个方向

广告搜索算法，本质上是在用户请求到来时，从海量候选中高效找出“既相关、又高收益、又体验可接受”的结果。

核心链路通常是：

Query / 用户请求
-> 召回 Recall
-> 粗排 Pre-rank
-> 精排 Rank
-> 机制层 Auction / Re-ranking / 约束控制
-> 展示
-> 点击 / 转化 / 收益反馈

## 面试先建立的大图

### 1. 搜索和推荐、广告的区别

- 搜索：用户意图显式，query 很关键，相关性优先。
- 推荐：用户意图隐式，依赖用户画像和行为序列。
- 广告：既要相关，也要收益，还要满足商业和策略约束。

一句话答法：

> 搜索更强调 query understanding 和 relevance，广告排序是在 relevance 之上再加入 CTR、CVR、bid、ROI、预算、频控等商业目标的多目标优化。

### 2. 广告搜索常见链路

- Query Understanding：分词、纠错、同义词扩展、类目理解、意图识别。
- Recall：关键词倒排、类目召回、向量召回、规则召回。
- Pre-rank：轻量模型快速筛候选。
- Rank：重模型预测 CTR/CVR/GMV/价值。
- Auction：结合 bid 和预估点击/转化价值出最终排序。
- Post-rank：去重、多样性、频控、业务规则。

## 高频知识点

## 1. 召回、粗排、精排分别做什么

- 召回：从海量候选里尽量别漏掉相关广告，重点是高召回、低时延。
- 粗排：用轻量模型快速缩小候选集。
- 精排：用更重模型做细粒度排序，优化最终线上指标。

常见追问：为什么不能直接精排全量候选？

答：因为计算成本和时延不允许。工业系统一定是分阶段漏斗，先便宜地筛，再昂贵地精排。

## 2. CTR / CVR / eCPM 分别是什么

- CTR：点击率，`click / impression`
- CVR：转化率，`conversion / click`
- eCPM：每千次展示收益，常见写法是 `bid * CTR * 1000`，更一般可写成 value-aware 形式

广告排序里最常见的回答：

> 排序不只看相关性，还会看预估 CTR、CVR、出价 bid，有的场景还会看 GMV、LTV、ROI 等长期价值。

## 3. GSP 和 OCPX / OCPA 是什么

- GSP：Generalized Second Price，广义二价拍卖。排序按价值排序，扣费接近下一名的临界价格。
- OCPX / OCPA：优化目标从点击扩展到转化或更深层目标，用转化概率做出价调整。

一句话答法：

> GSP 更偏点击广告和经典 sponsored search；OCPX 更强调按转化目标优化，让系统替广告主自动调节出价。

## 4. AUC、LogLoss、GAUC、NDCG、RPM 分别什么时候看

- AUC：二分类排序能力，常用于 CTR/CVR 模型离线评估。
- LogLoss：概率预测校准质量，更适合概率任务。
- GAUC：按用户/Query 分组后的 AUC，更接近实际排序场景。
- NDCG：看排序位置质量，搜索场景很常见。
- RPM / eCPM / Revenue：商业收益指标。

高频追问：为什么线上好不代表离线一定高？

答：因为离线样本分布、位置偏差、选择偏差、策略干预和线上机制层都可能让离线指标与真实线上收益不完全一致。

## 5. 广告排序为什么是多目标优化

因为目标至少有三类：

- 用户目标：相关性、体验、满意度
- 平台目标：收益、留存、生态健康
- 广告主目标：点击、转化、ROI、预算消耗

一句话答法：

> 广告排序不是单一预测问题，而是 relevance、CTR/CVR、bid、预算、频控、多样性等目标和约束共同决定的排序问题。

## 6. 为什么要做特征工程

典型特征分三类：

- 用户侧：画像、历史行为、短期兴趣、上下文
- Query / 内容侧：关键词、类目、意图、语义向量
- 交叉侧：user-ad、query-ad、user-query-ad 交叉特征

常见模型演进：

- LR / FM：简单、可解释
- Wide & Deep：记忆 + 泛化
- DeepFM / xDeepFM：显式/隐式特征交互
- DIN / DIEN：行为序列建模
- MMOE / PLE：多任务学习

## 7. 什么是多任务学习，为什么广告里常见

因为 CTR、CVR、CTCVR、跳失、停留时长等目标相关但不完全相同。

常见说法：

> 用共享底座 + 任务塔可以同时建模多个相关目标，提升样本利用率，但也要处理跷跷板效应和任务冲突。

## 8. 样本偏差问题

广告/搜索面试经常问的偏差：

- Position Bias：排得靠前天然更容易被点。
- Selection Bias：能被看到的数据不是全空间的随机样本。
- Delayed Feedback：转化反馈滞后。
- Data Shift：线上分布变化。

面试答法：

> 常见处理包括 IPS/DR、debias、校准、时间窗设计、负样本采样、在线实验和模型更新机制。

## 9. 校准 Calibration 为什么重要

因为广告系统不只是做排序，还会把概率拿去参与出价、预算和收益计算。CTR/CVR 如果不校准，会直接影响 auction 和 ROI。

关键句：

> 排序强不等于概率准，广告里 calibration 很关键，因为后续机制层真的会把预测概率当数值使用。

## 10. 冷启动怎么办

- 新广告：内容特征、类目特征、广告主特征、相似广告迁移
- 新用户：上下文特征、地域、设备、query 意图
- 新 query：倒排扩展、同义词、向量召回、规则 fallback

## 搜索算法方向必问点

## 11. 搜索相关性怎么做

经典三层：

- lexical matching：BM25、倒排、query-term overlap
- semantic matching：embedding、dual encoder
- interaction ranking：cross encoder / late interaction

你可以直接把自己的项目往这里映射：

> 我做过 query planning、多源召回、RRF 融合和 Cross-Encoder 精排，这一套思路和搜索排序体系是同构的，只是我的场景是学术检索，不是广告检索。

## 12. 搜索里的 Query Understanding 包括什么

- 分词
- 拼写纠错
- 同义词扩展
- 改写 rewrite
- 意图识别
- 类目理解
- 实体识别

如果面试官问“为什么 query understanding 重要”：

答：因为搜索上限往往由 query 理解和召回决定，排序只能在已有候选里调顺序，没召回进来的结果再强的 ranker 也救不回来。

## 13. Reranker 的价值和边界

价值：给候选做更细粒度语义排序。

边界：不能补召回。

这是你现有项目里最适合迁移的一句。

## 14. 线上系统为什么一定要分层缓存和低时延

因为广告/搜索系统是高 QPS、强时延约束系统。

常见点：

- 倒排索引 / ANN 索引
- 特征缓存
- embedding 缓存
- 结果缓存
- 异步特征更新
- 模型服务化与多级 fallback

## 广告机制常见追问

## 15. 排序分数一般怎么设计

非常常见的一句话：

> 一个简单但经典的排序目标是 `score = bid * pCTR`；如果更强调转化，就可能是 `bid * pCTR * pCVR * value` 或它的变体。

然后一定补一句：

> 真实系统里还会叠加相关性约束、预算约束、频控、探索、业务规则和重排策略。

## 16. 为什么不能只看收益最大化

因为会损伤用户体验和生态。

如果只追收益，可能出现：

- 低相关但高 bid 的广告占位
- 用户长期体验下降
- 广告主 ROI 失衡
- 平台生态恶化

所以真实系统常是“收益目标 + 体验约束 + 公平性/生态约束”。

## 17. 预算、频控、探索为什么重要

- 预算：控制广告主消耗节奏
- 频控：避免同一广告过度曝光
- 探索：避免系统只给历史强者流量，帮助新广告获得学习机会

这类问题核心是体现你理解系统不是“只训练模型”。

## 模型侧常见问题

## 18. LR 为什么在广告里经典

因为简单、稳定、可解释、上线成本低，对稀疏高维 one-hot 特征很有效。

常见回答：

> LR 在广告点击率预估里是工业界经典 baseline，因为训练快、可解释、特征工程空间大，而且容易和业务规则配合。

## 19. FM / DeepFM 为什么常见

因为广告场景中大量高维稀疏类别特征，需要显式或隐式建模特征交互。

## 20. DIN 为什么重要

因为用户兴趣不是静态的，要从历史行为序列里动态抽取与当前广告/商品最相关的兴趣片段。

## 21. MMOE / PLE 为什么会问

因为多目标任务很常见，面试官想看你是否理解：

- 任务共享带来的收益
- 任务冲突问题
- shared-bottom 到 expert-routing 的演化逻辑

## 22. 线上特征和离线特征不一致怎么办

这是工程常问点。

答法：

> 要保证 feature consistency，常见做法是统一特征平台、离在线同构、特征版本管理、回放校验和监控。

## 速记版答案

### 召回、粗排、精排

召回保上限，粗排控成本，精排定最终质量。

### CTR、CVR、eCPM

CTR 看点，CVR 看转化，eCPM 看收益。

### 为什么是多目标优化

因为广告同时服务用户、平台和广告主，不能只看一个指标。

### 为什么 Reranker 不够

因为它只能排已有候选，不能补召回。

### 为什么要漏斗架构

因为全量精排太贵，工业系统必须逐层筛选。

### GSP / OCPX

GSP 更经典点击拍卖，OCPX 更偏转化目标优化。

### 常见指标

AUC/GAUC 看排序，LogLoss 看概率，NDCG 看位次质量，RPM/eCPM 看收益。

### 偏差问题

广告数据不是自然随机采样，位置和曝光机制会引入强偏差。

### Calibration

因为预测概率会直接参与 auction 和收益计算，所以不仅要排得对，还要估得准。

### 项目迁移

我虽然没做广告业务，但做过 query understanding、召回、融合、精排和服务化，这些是广告搜索算法的主干能力。

## 最后建议

更稳的策略是：

- 承认自己业务机制层经验少
- 强调自己做过搜索/检索/排序主链路
- 把迁移能力讲清楚
- 把广告特有概念补到能答高频追问

一句话收尾：

> 我当前的强项在检索、排序和大模型系统，但广告搜索算法的核心主干和这些能力是相通的；我现在重点补的是 auction、多目标优化和广告业务指标，这部分我已经形成了完整的知识框架。
